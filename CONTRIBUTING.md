# Contributing to Sentinel Agents

We welcome all contributors. For security-sensitive components, you'll need:
1. 2fa enabled account
2. SignedCLA with crypto proof
3. Pass background check for core contributors

## Workflow
1. Find an issue labeled "good-first-issue"
2. Create a security audit PR for all changes
3. Get review by 2 core maintainers
4. Merge after passing all audits

## Security Process
All PRs require:
- ThreatModel.md documentation
- AttackSurface matrix
- Regression test covering changes
- Vulnerability report (if any)

## Code of Conduct
- Follow secure coding guidelines
- Report all security issues directly to security@sentinella
- No backdoors or privileged access in submitted code
